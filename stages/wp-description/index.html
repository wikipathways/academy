WikiPathways: Add a Description

Background

Each WikiPathways model needs to have a text description of the contents of the pathway, to improve its usability. The descriptions should be thorough and unique; it should NOT simply be a copy of text from another source. For example, the desciption should provide context and an explanation of the key parts of the pathway, as well as describe any figure legend, and any special glyphs, graphics or colors used. It is also a good idea to describe particularly complex parts of the pathway in detail, and to include a direct link to any published figure that the pathway was modeled after, even if the paper is included as a literature reference. For more information on what makes a good description, see the Ten simple rules for creating reusable pathway models for computational analysis and visualization paper.

Challenge

Add a description to your tutorial pathway at WikiPathways, or update the description you entered when you uploaded the pathway.


  • Launch PathVisio and open the pathway via file from the previous step via File > Open.
  • In the Properties tab on the right side, double-click the empty Comments field.
  • Enter the description in the input field and click Ok. You can copy and paste from a text editor.
  • save your work as a GPML file under File > Save As. We will upload this gpml to WikiPathways in the next step.
